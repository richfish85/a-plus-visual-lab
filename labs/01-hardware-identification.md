# Lab 01: Hardware Identification

## What

Identify common desktop and laptop hardware parts.

Focus parts:

- RAM
- storage drive
- motherboard
- CPU cooling
- power connector
- display cable
- USB ports
- Ethernet port

## Why

Hardware troubleshooting starts with knowing what you are looking at.

Example:

If a desktop powers on with no display, you need to recognize video cables, RAM slots, monitor input, and graphics connections.

## How

Use ticket `APLUS-001`.

Simplified pilot version:

- [APLUS-001: Desktop powers on, no display](../pilot-modules/aplus-001-desktop-powers-on-no-display.md)

Steps:

1. Read the ticket.
2. Point to the monitor, cable, and desktop.
3. Check the monitor input.
4. Check the video cable connection.
5. If using spare hardware, power off and unplug the desktop.
6. Identify RAM and graphics connection points.
7. Write the first safe check and likely cause.

Success condition:

```text
I can identify the likely hardware area and explain the first safe check.
```

Checklist:

- [ ] I identified the external display path.
- [ ] I identified RAM.
- [ ] I identified storage.
- [ ] I identified the power connection.
- [ ] I wrote the first safe check.

## Implementation

Materials:

- spare desktop or diagram
- labeled hardware photos
- cable cards
- ticket card
- safety checklist

Hands-on option:

Sort physical parts into trays:

```text
Memory -> Storage -> Power -> Display -> Network
```

Visual option:

Print or display a computer interior photo and label each part with sticky notes.

Checklist:

- [ ] Use physical parts if safe.
- [ ] Use diagrams if physical parts are unavailable.
- [ ] Keep labels visible.

## Assumptions

- The learner may not have spare computer parts.
- A diagram or photo can replace real hardware.
- The learner should not open powered equipment.

Checklist:

- [ ] Use the safe version first.
- [ ] Add physical practice only when ready.

## Threat/Risk Notes

- Do not open a power supply.
- Do not touch internal parts while powered.
- Do not force RAM, cables, or cards into slots.

Checklist:

- [ ] Power off.
- [ ] Unplug.
- [ ] Ground yourself.
- [ ] Handle parts by edges.

## Validation Steps

Evidence to save:

- labeled hardware photo or diagram
- ticket note for `APLUS-001`

Validation:

- [ ] The learner can name five parts.
- [ ] The learner can match each part to its purpose.
- [ ] The learner can explain one safe first check for no display.
