# Lab 04: Secure a Workstation

## What

Practice basic workstation security.

Focus items:

- phishing signs
- MFA
- password reset path
- local firewall
- user permissions
- safe communication

## Why

Security is part of everyday IT support.

Example:

A user reports a suspicious sign-in email. The safe action is to inspect signs, avoid the link, and use a trusted sign-in page.

## How

Use ticket `APLUS-004`.

Steps:

1. Read the suspicious email scenario.
2. Circle or list phishing signs.
3. Decide what not to click.
4. Write the safe user instruction.
5. Check MFA status in a dummy account or screenshot.
6. Document the response.

Success condition:

```text
I can identify phishing signs and give a safe next step.
```

Checklist:

- [ ] I did not click the link.
- [ ] I found three warning signs.
- [ ] I wrote the safe sign-in path.
- [ ] I explained MFA in plain language.
- [ ] I documented the ticket.

## Implementation

Materials:

- printed phishing sample or fake screenshot
- dummy account
- security station cards
- ticket card

Visual phishing signs:

```text
Urgency
Unknown sender
Mismatch link
Unexpected attachment
Request for password or code
```

Checklist:

- [ ] Use fake examples.
- [ ] Keep the signs visible.
- [ ] Practice the user-facing response.

## Assumptions

- No real phishing emails are needed.
- No real accounts are needed.
- The goal is safe recognition and response.

Checklist:

- [ ] Use dummy data.
- [ ] Avoid live risky links.

## Threat/Risk Notes

- Do not click suspicious links.
- Do not submit passwords into unknown pages.
- Do not forward suspicious content to others unless using approved reporting methods.
- Do not ask users for MFA codes.

Checklist:

- [ ] Report safely.
- [ ] Reset from trusted page only.
- [ ] Enable MFA where supported.

## Validation Steps

Evidence to save:

- annotated fake phishing sample
- ticket note for `APLUS-004`

Validation:

- [ ] The learner identifies three phishing signs.
- [ ] The learner writes a safe user instruction.
- [ ] The learner explains MFA in one sentence.
