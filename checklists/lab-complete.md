# Lab Complete Checklist

## What

Use this when a lab is finished.

Finished means the ticket has evidence, not just a fix.

## Why

Documentation is part of IT support.

It shows what happened, what changed, and what to try next time.

Example:

If the fix was "reseated RAM", the note should say what symptom changed after reseating it.

## How

Complete these steps:

- [ ] The ticket ID is recorded.
- [ ] The original problem is written in one sentence.
- [ ] The checks are listed in order.
- [ ] The fix is written clearly.
- [ ] The result is written clearly.
- [ ] Evidence is saved.
- [ ] Private data is removed.
- [ ] The ticket is moved to Done.

Ticket note template:

```text
Ticket:
Problem:
Visual clue:
Checks:
Fix:
Result:
Evidence:
Reflection:
```

Checklist:

- [ ] Write the result.
- [ ] Save evidence.
- [ ] Close the loop.
