# Session Start Checklist

## What

Use this checklist at the start of every lab session.

It creates the same beginning every time.

## Why

A predictable start lowers friction.

It also helps the learner know when to begin, what to touch, and when to stop.

Example:

Before opening a VM, the learner first checks the ticket ID, timer, materials, and success condition.

## How

Start here:

- [ ] I have one ticket only.
- [ ] I can see the lab goal.
- [ ] I know the success condition.
- [ ] I have the right tools open.
- [ ] I have a place to write notes.
- [ ] I set a 25 to 35 minute timer.
- [ ] I know what to do if I get stuck.

Stuck rule:

```text
If stuck for 5 minutes:
1. Stop changing things.
2. Write what you already checked.
3. Look at the diagram.
4. Try one next safe check.
```

Checklist:

- [ ] One task.
- [ ] One timer.
- [ ] One diagram.
- [ ] One result.
