# Reflection

## What

One-sentence learner reflection:

```text
I learned that connected Wi-Fi is only one part of reaching a website.
```

## Why

The reflection shows the practical lesson without turning the evidence into a long essay.

## How

Use this sentence pattern:

```text
I learned that [visible symptom] can still happen when [hidden cause] is wrong.
```

Checklist:

- [x] One sentence.
- [x] Plain language.
- [x] Linked to the ticket.
- [x] Safe to share.

## Implementation

This reflection belongs with ticket `APLUS-002`.

It supports the app prompt:

```text
Explain one ticket from symptom to done in one sentence.
```

## Assumptions

- The learner has completed the matching, topology, and ticket-board activities first.
- The reflection is written after the result is documented.

Checklist:

- [x] Reflection follows the ticket.
- [x] Reflection does not include private data.

## Threat/Risk Notes

Risk:

Reflections can become vague or inflated.

Response:

Keep the sentence concrete and tied to the actual action.

Checklist:

- [x] Do not overclaim.
- [x] Do not mention real systems.
- [x] Keep it practical.

## Validation Steps

- [x] The reflection is understandable.
- [x] The reflection fits the ticket.
- [x] The reflection is short enough for a portfolio note.
