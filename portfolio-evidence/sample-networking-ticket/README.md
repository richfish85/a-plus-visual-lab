# Sample Networking Ticket Evidence

## What

This is a sample evidence pack for a simulated CompTIA A+ networking ticket.

Ticket:

`APLUS-002`

Problem:

The device says Wi-Fi is connected, but websites do not load.

## Why

This example shows how the learner can turn one app activity into a portfolio artifact.

It keeps the evidence short, visual, and safe to share.

## How

Read the files in this order:

1. `ticket-note.md`
2. `topology-evidence.md`
3. `reflection.md`
4. `evidence-summary.txt`

Checklist:

- [x] Problem is named.
- [x] First checks are listed.
- [x] Result is clear.
- [x] Reflection is short.
- [x] No private data is included.

## Implementation

This sample uses dummy lab information.

The ticket path is:

```text
New -> Diagnose -> Fixing -> Document -> Done
```

Checklist:

- [x] Ticket note created.
- [x] Diagram note created.
- [x] Reflection created.
- [x] Evidence summary created.

## Assumptions

- The network is a lab or simulator.
- The learner is not changing a real home, school, or work router.
- The goal is to practice troubleshooting order.

Checklist:

- [x] Use simulated data.
- [x] Avoid real account details.
- [x] Keep the action narrow.

## Threat/Risk Notes

Risk:

Real network screenshots can expose private network names or addresses.

Response:

This sample uses no screenshots and no real addresses.

Checklist:

- [x] No real SSID.
- [x] No real IP address.
- [x] No real username.
- [x] No real device serial number.

## Validation Steps

- [x] The ticket has one problem.
- [x] The action matches the problem.
- [x] The result is measurable.
- [x] The evidence is safe to share.
