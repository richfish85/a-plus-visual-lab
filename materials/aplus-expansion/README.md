# A+ Expansion Plan

## What

This folder is the planning stash for scaling the technician field manual toward complete CompTIA A+ coverage.

It is not the public app yet. It keeps the larger content plan separate so the current portfolio demo can stay clean.

## Why

The field manual should prove practical support judgement, not become a copied textbook.

The content model is:

1. Start with the official A+ domains.
2. Write a boiled-down technician summary for each domain.
3. Keep deeper links separate from the main material.
4. Identify realistic support scenarios.
5. Promote only the strongest scenarios into visual modules.

## How

Files:

- [core-1-map.md](core-1-map.md) maps 220-1201 domains to technician outcomes.
- [core-2-map.md](core-2-map.md) maps 220-1202 domains to technician outcomes.
- [domain-summaries.md](domain-summaries.md) gives the plain-English summary layer.
- [scenario-candidates.md](scenario-candidates.md) lists 3-6 likely field-manual scenarios per domain.
- [reference-links.md](reference-links.md) keeps official and deeper links out of the main content.

## Content Rules

- Keep each summary short.
- Use technician language, not exam-cram language.
- Put links at the bottom or in the reference file.
- Treat references as support, not the main learning path.
- Do not add CRUD until static editing becomes painful.
