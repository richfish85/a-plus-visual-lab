1 / What happened

An application opens, then closes, freezes, or shows the same error each time.

Symptoms
Launch / crash / repeated error
Applies to
One installed desktop app
First rule
Capture exact error and protect user data first
2 / Why it happens

Common causes are one-user settings, damaged app files, low disk space, or a recent update.

3 / Diagnose

Is it only one user?

  1. Capture exact error.Screenshot or copy the wording before trying repairs.
  2. Check scope.Ask whether other users or devices have the same problem.
  3. Try a safe test profile if allowed.Do not use another user's account.
  4. Check recent setting changes.Record what changed and when it started.
3 / Diagnose

Is the app install damaged?

  1. Restart app and device.Confirm the issue repeats after a clean restart.
  2. Check disk space.Low disk can break app launch and updates.
  3. Check supported repair options.Use vendor-supported repair only.
  4. Check logs.Record Event Viewer, app log, or crash code.
3 / Diagnose

Did an update cause it?

  1. Ask when it started.Compare start time with updates or installs.
  2. Check update history.Record Windows and app updates that match the timing.
  3. Check app version.Record installed version and expected version.
  4. Check service status.Cloud apps may fail during vendor outages.
4 / Fix safely

Repair user-level issue

Do not delete user data, profiles, or app folders without backup and approval.
  1. Restart app and device.
  2. Reset app preferences only if approved.
  3. Clear app cache only if supported.
  4. Escalate if profile corruption is suspected.
  5. Verify: same user can open the app and use core functions.
4 / Fix safely

Repair application install

Check business impact before repairing or reinstalling shared apps.
  1. Back up or confirm user data location.
  2. Use app repair if available.
  3. Install approved update.
  4. Reinstall only if authorised.
  5. Verify: app opens after restart and user data remains present.
4 / Fix safely

Handle update-related crash

Do not roll back updates unless policy allows.
  1. Install approved app patches.
  2. Restart after update.
  3. Check vendor status or known issues.
  4. Escalate with update, version, and error details.
  5. Verify: launch succeeds or escalation has evidence.
5 / Document it

Record exact error, scope, checks, action, and data safety.
