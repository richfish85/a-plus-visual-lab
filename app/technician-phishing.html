Back to runbook
1 / What happened

The user received a message asking them to sign in, verify an account, or act urgently.

Symptoms
Urgent message / unknown sender / login link
Applies to
Email, chat, or browser sign-in warnings
First rule
Do not click the link or enter credentials
2 / Why it happens

The message may be phishing, a compromised sender, or a real alert that must be checked through a trusted path.

3 / Diagnose

Does it show phishing signs?

  1. Inspect sender.Look for strange domains, misspellings, or an unexpected sender.
  2. Check the request.Password, MFA code, payment, or urgent threat is high risk.
  3. Check link text safely.Use preview or security tools only. Do not open the link.
  4. Ask what the user did.Confirm whether they clicked, entered a password, or approved MFA.
3 / Diagnose

Did the user enter credentials?

  1. Ask calmly.Find out if password, MFA code, or approval was entered.
  2. Record timing.Write down when the action happened and from what device.
  3. Check account alerts.Use approved admin or security tools only.
  4. Escalate immediately.Security may need to reset sessions and review sign-ins.
3 / Diagnose

Could it be real?

  1. Ignore the email link.Open the service from a trusted bookmark.
  2. Check official account notifications.Look inside the trusted service.
  3. Confirm with policy.Some services send real security alerts.
  4. Document uncertainty.Do not overstate if not confirmed.
4 / Fix safely

Contain the phishing report

Never ask for passwords, MFA codes, or screenshots containing secrets.
  1. Tell user not to click, reply, or forward casually.
  2. Report via approved phishing process.
  3. Verify account from a trusted sign-in page.
  4. Quarantine or delete only if policy allows.
  5. Verify: no credentials or MFA approvals were given.
4 / Fix safely

Protect the account

Treat entered passwords, MFA codes, and unexpected MFA approvals as urgent.
  1. Start trusted password reset.
  2. Escalate for session review and MFA review.
  3. Preserve evidence of the message.
  4. Check for suspicious inbox rules if policy allows.
  5. Verify: security review or escalation is recorded.
4 / Fix safely

Respond through trusted path

Do not use the email link even if it may be real.
  1. Open service from trusted URL.
  2. Check account security page.
  3. Complete required action inside trusted service.
  4. Report suspicious copy if needed.
  5. Verify: user understands safe path.
5 / Document it

Record user action, advice given, and escalation status.
