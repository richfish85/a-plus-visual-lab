Back to runbook
1 / What happened

A Windows virtual machine is slow and Windows reports low disk space.

Symptoms
Slow boot / red storage bar / updates may fail
Applies to
Windows VM or safe lab machine
First rule
Snapshot or confirm restore path before cleanup
2 / Why it happens

Most cases are low free space, too many startup tasks, or updates stuck because space is low.

3 / Diagnose

Is disk space the bottleneck?

  1. Open Storage settings.Record free space in GB and percent.
  2. Check temporary files.Use Windows cleanup categories. Do not browse-delete system folders.
  3. Check host storage.A VM can slow down if the host disk is nearly full.
  4. Check VM snapshots.Old snapshots can grow large and slow the VM.
3 / Diagnose

Are startup apps slowing boot?

  1. Open Task Manager.Check CPU, memory, disk, and Startup apps.
  2. Sort by startup impact.Look for high-impact non-essential apps.
  3. Ask what changed.Recent installs, agents, or updates can explain new slowness.
  4. Restart and time it.Confirm the symptom is repeatable before changing settings.
3 / Diagnose

Are updates pending or stuck?

  1. Open Windows Update.Record pending, failed, or paused updates.
  2. Check restart requirement.Some updates wait for reboot before cleanup helps.
  3. Check free space.Low disk space can block update download and install.
  4. Record error codes.Escalation needs exact update name and error text.
4 / Fix safely

Free safe storage

Do not delete unknown system folders, user files, or security tools.
  1. Take a VM snapshot or confirm restore path.
  2. Use Windows Storage cleanup for temporary files.
  3. Empty Recycle Bin only if approved.
  4. Escalate if the VM disk needs resizing.
  5. Verify: free space improves and VM restarts.
4 / Fix safely

Reduce startup load

Do not disable antivirus, backup, remote support, or management tools.
  1. Disable only clearly unnecessary startup apps.
  2. Restart and time startup again.
  3. Document every disabled item.
  4. Re-enable anything that causes problems.
  5. Verify: startup improves without losing required functions.
4 / Fix safely

Complete updates safely

Do not interrupt updates while they are installing or restarting.
  1. Free enough disk space first.
  2. Install pending updates.
  3. Restart when prompted.
  4. Record failed update codes if unresolved.
  5. Verify: update page shows current status or a known error.
5 / Document it

Record before/after state and every changed setting.
